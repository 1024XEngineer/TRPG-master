# MVP架构设计草案

> 定位：对 `AI-DM系统_MVP架构设计_v0.1.md` 的客户端层与交互设计进行移动端重构。服务端（Agent / Tool / DB / 管线 / 记忆）不做结构性修改，仅增加移动端适配的消息与状态。
> 前提：每个玩家用**手机竖屏**玩游戏，不是电脑浏览器。
> 版本：v0.1 · 日期：2026-07-09

---

## 0. 移动端 vs PC 网页：这不是"缩放"，是范式切换

| 维度 | PC 网页版 (原设计) | 手机版 (修正) |
|---|---|---|
| 屏幕 | 宽屏，可多面板并排 | 竖屏，单列，一次只看一件事 |
| 输入 | 键盘打字 | 拇指输入 + **快捷动作按钮**（MS2 加语音） |
| 注意力 | 可同时看聊天+角色卡+骰子记录 | **叙事流为主，其他切 Tab 或弹层看** |
| 网络 | 稳定 WiFi | 4G/5G 不稳定，切后台断连 |
| 交互 | 鼠标悬浮、右键 | 长按、滑动、底部拇指区 |
| 通知 | 网页通知（弱） | 推送通知（"到你了"） |

## 1. 客户端技术选型修正

### 1.1 选型结论：React Native（或 Expo）

| 选项 | 判据 | 结论 |
|---|---|---|
| React Native / Expo | TS 共享类型、一套语言、热更新、两端发布 | **推荐** |
| Flutter | 性能好但 Dart 生态与后端 TS 割裂 | 备选 |
| 移动 H5 (PWA) | 最快但体验始终差原生一档（键盘弹出、手势冲突、切后台重连） | 不做主方案 |
| 微信小程序 | 国内分发优势大，但 WebSocket 长连接受限、Tool Calling 延迟不可控 | 可做 **小程序壳 + H5 内嵌**的折中 |

**MVP 推荐路线**：**Expo (React Native) 直接出 iOS + Android**。理由三条：
1. `shared/types/` 目录的 TS 类型前端内部共享零成本；后端 Python 侧通过 OpenAPI / JSON Schema 做类型契约同步；
2. Anthropic Python SDK 原生支持 Tool Calling，服务端调 LLM 不依赖 Node.js 生态；
3. Expo 的热更新能力意味着修 Agent 行为不需要发版审核。

如果你们渠道侧重微信生态，走**小程序 WebView 内嵌移动 H5**——但 WebSocket 长连接在小程序里有限制，需要加 HTTP 轮询降级。这个决策看分发策略，纯技术上 Expo 最干净。

### 2.2 交互分层

手机屏只能同时展示一个"关注面"，所以交互分成三层：

| 层 | 常态 | 触发方式 | 内容 |
|---|---|---|---|
| **主层** | ✅ 始终可见 | — | 叙事流 + 快捷动作 + 输入栏 |
| **弹出层** | ❌ 隐藏 | 点顶部角色头像 | 角色卡半屏弹层 (Bottom Sheet) |
| **弹出层** | ❌ 隐藏 | 点顶部线索图标 | 线索手账半屏弹层 |
| **弹出层** | ❌ 隐藏 | 点掷骰卡片 | 骰值详情弹层 |
| **弹出层** | ❌ 隐藏 | 长按叙事文本某条 | 追问/复述/标记可疑 |
| **切换层** | ❌ 隐藏 | 点底部私密 Tab | 私密频道（你和 DM 的密语） |

**关键设计**：玩家在弹层里只能**看**信息，不能操作——发行动必须回到主层输入栏。这防止多人等一个人翻包。

### 2.3 快捷动作系统（移动端核心创新——PC 版没有的）

手机上打字描述"我要搜索书房的抽屉"太慢了。快捷动作系统把 TRPG 最高频的 8 个动作用**按钮**表达，点击后自动填入输入框或带上下文直接发送：

```
┌─────────────────────────────────────────────┐
│  🔍 搜索  │  💬 交谈  │  👀 观察  │  📖 阅读  │  ← 可横滑
│  🏃 移动  │  🗣 说服  │  🔧 使用  │  ⚔️ 自卫  │
└─────────────────────────────────────────────┘
```

点击后行为：
- 点击 `🔍 搜索` → 输入框自动填入 `"我搜索"`，光标定位在"搜索"后面，玩家补对象（"书房"）即可发送
- 点击 `💬 交谈` → 弹出当前场景 NPC 列表，选人后自动填入 `"我和[管家]交谈"`
- 长按 `🔍 搜索` → 弹出该角色所有技能列表，**点技能直接发检定请求**
- 自定义：玩家可把自己常用句式存成快捷动作（v1）

**这 8 个按钮把"打字 15 个字"变成"点 2 下"，是手机跑团体验的及格线。**

### 2.4 叙事流的视觉设计

```
常规叙事文本:
  白色/浅灰主色，16px，行高 1.6
  每段之间空行分隔
  "环境描写"用斜体
  NPC 台词用引号 + 该 NPC 专属颜色标记

掷骰结果卡片:
  深色半透明底，嵌在流中
  显示: 🎲图标 + 技能名 + 骰值 vs 目标 + 成功等级条
  成功=绿色描边，失败=红色描边，大成功=金色光效

系统消息:
  居中细条，小字灰色
  "📍 场景：老宅书房" / "⏰ 时间流逝 30 分钟"

恐怖氛围文本:
  SAN 检定触发的心理描写 → 文字逐渐变暗/变红
  关键恐怖揭示 → 屏幕轻微震动 + 短暂黑屏闪烁 (可选，可关)
```

### 2.5 通知与"到你了"

跑团不像即时聊天——玩家可能放下手机等 DM 描述完。移动端需要：

1. **DM 叙事结束且轮到该玩家时 → 推送通知**："守秘人等待你的行动"（可开关）
2. **DM 直接 @ 你时 → 推送通知**
3. **SAN 检定 / 受伤 → 推送通知 + 手机震动**
4. **房间内"打字中"指示器**：看到其他玩家头像旁有"..."就知道有人在输入

## 3. 服务端需新增的移动端适配（改动极小）

### 3.1 消息协议增加移动端专用字段

```typescript
// shared/types/messages.ts 新增

type ServerMessage =
  // ... 原有字段不变 ...
  | { type: 'TURN_READY';     characterId: string; characterName: string }
    // 告诉客户端"到你了"，触发推送通知
  | { type: 'TYPING_INDICATOR'; characterId: string }
    // 某个玩家正在输入
  | { type: 'VIBRATION';      pattern: 'SAN_LOSS' | 'DAMAGE' | 'WHISPER' }
    // 触发手机震动
  | { type: 'QUICK_ACTIONS';  actions: QuickAction[] }
    // DM Agent 根据当前场景上下文，下发动作用推荐列表
    // 例如在书房场景 → [{id:'search', label:'搜索房间', skill:'侦查'},
    //                    {id:'read', label:'翻阅书籍', skill:'图书馆使用'}]
```

### 3.2 DM Agent 增加场景感知的动作推荐

DM Agent 的 System Prompt 增加一条输出指令：

```
§9 快捷动作推荐（移动端专用）
  每次场景切换或玩家进入新地点时，输出一组推荐动作。
  格式: { quickActions: [{id, label, skill?}] }
  限制: 不超过 4 个，匹配当前场景可能性最高的行动。
```

这样玩家进入书房时，快捷动作栏不只是固定的 8 个——DM 会动态推荐"搜索书房 / 翻阅书籍 / 查看窗外 / 检查壁炉"。

### 3.3 WebSocket 断线重连 + 消息补发

手机端 4G/5G 切换、电梯、地下室、切后台——断连是常态。

```python
# server/gateway/ws_server.py 新增
# 连接管理:
# - 每连接维护 lastReceivedSeq: int
# - 断线时不清除会话，标记 connection_status='RECONNECTING'
# - 30秒内重连 → 补发 events 表中 seq > lastReceivedSeq 的所有事件
# - 超时 → 角色原地挂机，DM Agent 被告知"该玩家暂离"
#
# client-mobile/lib/wsClient.ts
# - AppState 监听 (active/background)
# - 切后台 → 保持心跳 30 秒
# - 回前台 → 若断连则自动重连 + 请求补发
```

## 5. 技术选型总表（修正后）

| 层 | 修正前 (PC版) | 修正后 (手机版) | 理由 |
|---|---|---|---|
| 客户端 | React SPA | **Expo (React Native)** | 手机原生体验 + TS 共享 |
| 状态管理 | Zustand | Zustand（不变） | 客户端轻量状态 |
| WebSocket | `ws` (浏览器的) | React Native 的 WebSocket + 断线重连库 | 原生 WebSocket 支持 |
| 推送 | 网页通知 | **FCM (Android) + APNs (iOS)** | 手机推送是刚需 |
| 语音输入 | 无 | **MS1 不做，文字输入为主** | MS2 再做本地 STT 语音输入 |
| 氛围 | 网页 BGM | React Native 音频库 (expo-av) | 可选，非 MVP 核心 |
| 后端 | Node.js + Prisma | **Python FastAPI** | 后端组技术栈统一 |
| DM Agent | Claude SDK (Python) | **不变** | Anthropic Python SDK 原生支持 Tool Calling |
| DB | PostgreSQL | **不变** | — |
