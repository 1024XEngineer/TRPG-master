> 修订日期：2026-07-23

> 结论性质：样本统计与 Capability Gap 证据，不是 Schema 设计或 Contract 决议
> 文档归属：Module Parser 设计输入

## 一、结论边界

本报告回答以下问题：

1. 每个样本模组包含哪些概念。
2. 哪些归一化概念在 15 个样本中均可识别。
3. 哪些概念只在部分样本中出现。
4. 哪些机制可能抽象为统一的运行能力。
5. 为保真覆盖这批样本，ModuleContent 至少需要能够承载哪些信息。

本报告不直接证明：

- 某个概念应成为独立一级模型。
- 某个字段应成为逐模组必填字段。
- 某项机制必须由 Runtime 自动执行。
- 当前 `module-content.schema.json` 应立即增加对应字段。
- 这 15 个样本能够代表全部 CoC 模组。

需要区分四个不同层次：

1. **正文存在**：纸面模组中出现了某类内容或机制。
2. **Contract 可表达**：共享数据模型能够保真承载该内容。
3. **Contract 结构化**：该内容拥有独立、可验证的数据结构。
4. **Runtime 自动执行**：软件能够自动判断条件、改变状态和结算结果。

本次统计直接支持第 1 层，并可为后续三层提供候选证据，但不能单独完成字段准入或 Runtime 决策。

## 三、统计口径

### 3.1 编码方式

- `E`（Explicit）：原文存在可定位的明确表述、段落、列表、表格或章节。
- `N`（Normalized）：原文没有把它命名为独立概念，但可从连续流程中归一化识别。
- `—`：未发现满足当前定义的内容。

共同概念的覆盖数同时报告：

- `E`：明确表达数。
- `E+N`：归一化后可识别数。

### 3.2 伞形概念采用 OR 口径

例如：

- “场景前提”包含时代、地点、初始处境或事件前提；具备其中至少一种即可计入。
- “可游玩内容段”可以表现为场景、地点、任务、事件、章节或连续解谜段。
- “行动触发”可以表现为检定、时间、位置、持有物、玩家选择或既有状态。
- “终局”只要求存在可识别的最终后果，不要求具有多个结局。

因此，某个伞形概念为 15/15，不代表其所有子类型都应成为必填内容。

### 3.3 标签可以重叠

同一个 NPC 可以同时是行动主体、可交互对象和信息来源；同一个结果也可以同时是普通效果和终局后果。本文统计的是语义标签，不是互斥的数据表。

## 五、15 个样本的归一化语义交集

### 5.1 概念定义

| 归一化概念 | 纳入标准 | 不由该统计推出的内容 |
|---|---|---|
| 模组身份 | 至少有标题或可识别名称 | 不推出作者、版本、人数、时长全部必填 |
| 场景前提 | 存在时代、地点、初始处境或核心事件之一 | 不推出每个子项都必须独立结构化 |
| 幕后信息 | 存在真相、秘密、异常原理或隐藏因果 | 不推出必须有独立“真相”章节 |
| 进入方式 | 调查员有可识别的参与原因或初始状态 | 不推出只能使用委托式导入 |
| 可游玩内容段 | 存在场景、地点、任务、事件、章节或连续解谜段 | 不推出固定三幕式或统一章节类型 |
| 参与／交互对象 | 存在 NPC、怪物、物品、组织成员或环境对象 | 不推出所有对象都需要完整属性块 |
| 可获得信息 | 存在线索、对话信息、文件、观察结果或记忆 | 不推出所有信息都是“线索实体” |
| 行动触发 | 存在检定、选择、位置、时间、持有物或状态条件 | 不推出所有自然语言条件都能直接机器求值 |
| 行动后果 | 行动会揭示信息、改变处境、造成数值或叙事后果 | 不推出所有后果都应转成 Operation |
| 终局结果 | 存在成功、失败、逃离、留下、揭露或其他最终后果 | 不推出必须有多个结局或胜利条件 |

## 八、统一运行能力候选及职责边界

纸面模组可以由 KP 手工执行。正文存在某种机制，不代表软件必须自动执行。若产品目标要求自动化，样本提示可以继续研究以下能力。

| 机制 | ModuleContent 可声明的静态内容候选 | GameState／Runtime 的动态责任 | 其他边界 |
|---|---|---|---|
| 内容导航 | 内容段、入口、可达或触发关系 | 当前内容段、已进入内容、导航执行 | Validation 校验引用和不可达内容 |
| 信息揭示 | 事实、线索、来源、揭示条件 | 谁在何时获得了哪些信息 | Projection 负责防止向错误角色泄漏 |
| 条件与后果 | 可判定条件、声明式结果 | 求值条件并执行支持的状态变化 | 自然语言裁量不能自动视为可执行表达式 |
| 状态轨道 | 阶段、阈值、转换规则、阶段效果 | 当前值、当前阶段、转换历史 | Validation 校验阶段和转换完整性 |
| 时间调度 | 时间表、日程、到期触发定义 | 当前时钟、已触发事件、调度执行 | 不要求所有模组采用统一全局时钟 |
| 随机表 | 表项、骰式或抽取方式 | 投掷／抽取结果和已用记录 | 样本未证明必须支持权重、去重等额外语义 |
| 分支与终局 | 分支条件、普通后果、终局定义 | 已做选择、当前分支、最终结果 | — |
| 势力反应 | 势力、初始关系、静态反应规则 | 当前态度、资源、已执行行动 | — |
| 空间关系 | 地点、连接、隐藏条件 | 当前角色位置、已发现路线 | Projection 控制未发现地点的可见性 |
| 遭遇入口 | 参与者、入口条件、退出／结束条件 | 当前回合和参与者状态 | 战斗、追逐、SAN、技能算法属于 Ruleset |
| 特殊物品 | 定义、可用动作、条件和效果声明 | 持有者、数量、消耗及当前状态 | 通用装备算法属于 Ruleset |
| 跨模组引用 | 依赖、插入点、兼容声明 | 装载后的有效内容集合 | 主要属于 Package／Loader／Repository |

另外：

- 关键线索死锁检查属于 Validation／Review，不是 Runtime 运行职责。
- Keeper／Player／角色级可见性可以在内容中声明，实际过滤属于 Projection。
- 初始状态可以来自内容，开局后的当前状态只属于 GameState。
- 普通危险、战斗、追逐、谜题和社会冲突不能仅凭本轮统计强行归并为同一种 Encounter 执行语义。

## 九、样本对 ModuleContent 目标表达范围的启示

> 本章所称“目标表达范围”仅指根据 15 个样本得到的最低表达需求候选，不代表当前 v1 Schema、一级模型、必填字段或已经支持的能力。

### 9.1 15/15 样本支持的最低表达范围候选

若目标是保真覆盖这 15 个样本，ModuleContent 应能表达或承载：

1. 模组身份。
2. 场景前提。
3. 幕后信息。
4. 调查员进入方式。
5. 显式或隐式的可游玩内容段。
6. 参与者和可交互对象。
7. 可获得信息及其来源。
8. 行动、选择或其他触发因素。
9. 普通结果和长期后果。
10. 终局结果。

这十项是表达范围，不是十个已经获准的顶层字段。

### 9.2 为完整覆盖样本而出现的可选需求

如果目标不是只覆盖共同交集，而是保真覆盖全部 15 个样本中的特殊机制，还需要为下列内容提供可选表达方式，或明确记录 Capability Gap：

- 地图和空间关系。
- 角色准备模板。
- 阶段或计数轨道。
- 时间表、日程和倒计时。
- 随机事件表。
- 线索网络。
- 角色级秘密。
- 势力反应。
- 跨模组引用。
- 模组特有规则和特殊物品效果。

是否将它们结构化、放入哪个版本、由哪个消费者执行，不能只由出现频次决定。

### 9.3 本样本不能支持其成为逐模组必填项的内容

由于没有达到 15/15，本轮统计不能支持以下内容成为每一个模组都必须填写的项目：

- 推荐技能。
- 完整人数、时长、时代和规则版本元信息。
- 正式地图文件。
- 预制角色或 HO。
- 时间线或全局时钟。
- 势力系统。
- 随机表。
- 感染值、异化值等特定轨道。
- 线索网络。
- 跨模组组合关系。
- 多个结局。
- 战斗遭遇。
- 法术或神话实体。

低频只能否定“逐模组必填”的统计依据，不能据此否定这些概念作为可选能力进入某个版本。

### 9.4 正式字段准入仍需其他证据

样本概念进入正式 Contract 前，仍需回答：

- 谁生产该数据。
- 谁消费该数据。
- 在哪个时点消费。
- 忽略后是否影响模组正确运行。
- Parser 是否能稳定提取。
- Validation 是否能验证。
- Runtime／Projection 是否存在确定消费者。
- 是否有端到端测试和迁移策略。

## 十、最终结论

在桌面目录的这 15 个 CoC 样本中，模组身份、叙事前提、幕后信息、进入方式、可游玩内容段、参与对象、可获得信息、触发因素、行动后果和终局结果均可识别。

因此，如果 ModuleContent 的目标是保真覆盖这批样本，它至少应当能够承载这些信息；本统计不证明它们必须分别成为一级模型、必填字段或自动执行的 Runtime 机制。

阶段轨道、时间调度、随机表、角色级秘密、势力反应、线索网络和跨模组引用是部分样本提出的可选能力需求。它们应由通用声明表达、专用 Runtime 自动执行，还是暂时记录为 Capability Gap，需要结合确定性消费者、验证能力和端到端证据另行决定。
