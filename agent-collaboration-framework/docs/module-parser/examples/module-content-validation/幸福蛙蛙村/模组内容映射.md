## 模组简介

幸福蛙蛙村是一个带有温馨与诡异双重色彩的短模组，适合 1-4 名调查员。核心机制包括：多导入入口、软判据说服、累积变异阶段推进、SAN 检定、多结局选择。使用 CoC 7 版规则。

## E. 结论

### Contract 是否需要修改？

**发布设计层面不需要。** 幸福蛙蛙村的所有结构化内容均可被本样例实际使用的
5 个顶层集合（发布契约共 6 个）+ 四张表表达。发现的 18 个 Gap 全部属于
Parser/Runtime/Host Agent/Ruleset/Future Capability 层，没有暴露新的领域对象需求；
声明字段已进入当前 B/C Contract，但不表示占位 Runtime 已实现全部执行语义。

### 关键验证点

1. **软判据**（`difficulty=None`）——P0 的关键设计在此模组中得到了完美的真实用例验证。
2. **Track 退化为 Rule 组合**——四级变异阶段推进完全可通过 `Rule(on_state_change)` + 内建变量表达，不需要独立 TrackSpec。
3. **多结局**——5 个结局通过 `WinConditionSpec(is_ending)` 清晰区分。
4. **多入口**——3 种导入通过 `Entity.state + Rule(on_scene_enter)` 表达。
5. **复杂因果链**——"伤害青蛙→融合巨型青蛙怪"、"詹姆斯戒断→自杀"通过 `Rule + trigger_rule` 组合表达。

### 模组独特性

幸福蛙蛙村最独特的机制是**软判据说服信使**（P0 `difficulty=None` 的目标用例），以及**四级累积变异 Track**（P2 `on_state_change` + 内建变量）。两者均在当前 P2 Contract 中有明确表达路径。
