# 产品原型 —— Arkham Case Files · AI 守秘人

> **文档定位**：本项目的**产品原型指南（活文档）**——把已成形的 app 交互原型沉淀成可追溯的文字规格，作为后续设计、架构、开发对齐的**共同参照物**。


## 零、一句话

一个**面向多种桌游的 AI 主持人**平台，**当前阶段只落地 COC（克苏鲁的呼唤）**，但世界观、模组、玩法在架构上都要抽象、可扩展。玩家用手机各自加入房间，选 COC → 选模组，AI 守秘人知晓全部信息，玩家以**自然语言输入行动**，AI 负责旁白、事件触发、规则裁决与剧情推进，带完整一场直到模组通关。

原型代号 **Arkham Case Files**（阿卡姆案件档案），首发内置案件《**古宅幽影 / The Shadow Over Arkham**》。

## 一、玩家游玩形式（用户口述）

| 环节 | 描述 |
|---|---|
| 加入 | 每位玩家打开 app，进入同一房间 |
| 选择 | 选择游戏 **COC** → 选择**模组** |
| 开局 | 全员就绪后开始游玩 |
| 主持 | AI 守秘人**知晓全部信息**（身份/线索/答案/隐藏状态），负责剧情处理、事件触发、规则裁决 |
| 输入 | 每位玩家**分别输入自己的行动**，**以语音为主、文字为辅**（模拟真实 COC 桌面口述，见第五节 D1） |
| 结束 | 一路推进直到 COC 模组剧情**完成通关** |

## 二、多游戏定位与架构预留（重要）

产品是**多游戏的 AI 主持人货架**，COC 只是第一个上架的主持人。虽然本期只做 COC，但**架构与功能层面都要为扩展预留**：

| 游戏 | 原型状态 | 说明 |
|---|---|---|
| **跑团（DND / COC）** | 推荐 · COC 已就绪 | 本期唯一落地方向 |
| 狼人杀 | 开发中（占位） | 社交推理 |
| 血染钟楼 | 开发中（占位） | 更复杂的社交推理 |
| 剧本杀 | 开发中（占位） | 沉浸式角色扮演推理 |

**抽象分层思路（偏后端）**：
- **世界观层**：把 COC 的世界观（1920s 恐怖调查、7th 版规则、属性/技能/理智体系）抽象为一套可复用的规则世界。
- **模组层**：在世界观之上，每个**具体模组**（如《古宅幽影》）再抽象为独立的剧本数据（场景/线索链/NPC/事件/胜负条件）。
- 这套「世界观 → 模组」两级抽象，未来替换为其它游戏时，只需替换对应的世界规则与内容包。

> **先做单模组、不过早抽象通用框架**（过早抽象是这类项目第一死因）；此处「预留」指接口/数据结构上不写死 COC，而非现在就把多游戏框架实现出来。

## 三、界面流程（7 屏主线）

```
S1 登录 ⟹ S2 选游戏 ⟹ S3 选世界 ⟹ S4 剧情引入 ⟹ S5 创建调查员 ⟹ S6 房间大厅 ⟹ S7 游戏聊天（核心）
```

| # | 屏 | 关键元素 | 备注 |
|---|---|---|---|
| S1 | **登录** | 扫码加入房间 / 创建新房间 / 浏览已有案件；品牌「Arkham Case Files · AI 守秘人」；标注「COC 7th 相容」 | 强调「扫码上车 · AI 主持 · 即开即玩」 |
| S2 | **选游戏** | 跑团（推荐）/ 狼人杀 / 血染钟楼 / 剧本杀 | 仅「跑团」可点，其余标「开发中」 |
| S3 | **选世界** | 龙与地下城 5e / **克苏鲁的呼唤 7th（推荐·已就绪）** | 仅 COC 可进入；「更多世界设定开发中」 |
| S3.5 | **选模组**（待补屏） | 内置模组 + **玩家自导入模组** | 原型暂缺此屏，见第五节 D2 |
| S4 | **剧情引入** | 案件档案 `#1927-03`《古宅幽影》；暗色沉浸式开场白（惠特利教授失踪、蓝绿色光） | 内置测试模组，供全流程验证 |
| S5 | **创建调查员（车卡）** | ①选职业（侦探/记者/教授/医生/警察/古董商）②基本信息（姓名/年龄/出生地）③八大属性（STR/CON/SIZ/DEX/INT/POW/APP/EDU） | **MVP 必做**；4 步进度条；可存草稿 |
| S6 | **房间大厅** | 房间码 `AR-1927`、玩家列表与就绪状态、开始游戏、修改角色 | ~~AI 自动补位~~ **不纳入 MVP**（见 D4） |
| S7 | **游戏聊天（核心）** | 见第四节 | 全部游玩发生在此 |

## 四、核心游戏界面 S7 拆解

游戏主循环发生在一个**聊天式界面**里，围绕「守秘人旁白 ↔ 玩家行动 ↔ 掷骰裁决 ↔ 线索产出」运转。

### 4.1 消息流
- **AI 守秘人**：旁白（斜体·左侧竖线强调）、行动询问、要求技能检定、裁决结果、线索发放。
- **玩家消息**：自己的行动（自然语言）、掷骰结果。
- **系统消息**：如「案件档案已加载 · AI 守秘人在线」。
- **打字指示器**：AI 生成中的三点动画。

### 4.2 掷骰区
- 快捷骰：`D%`（百分骰，COC 核心）、`D20`、`D6`。
- 显示当前待检定技能（如「侦察 65%」），掷骰结果自动判定**成功/失败**并回填消息流。

### 4.3 底部功能面板（4 个抽屉）
| 面板 | 内容 |
|---|---|
| **角色卡** | 职业/年龄/出生地、HP、SAN（理智）、幸运、八大属性、装备清单 |
| **技能** | 技能条形图（侦察/聆听/图书馆/说服/斗殴/潜行/心理学/急救/神秘学…） |
| **地图** | 地图渲染区 + 已知地点列表（当前位置高亮，如正门/前院/门厅/书房/二楼/地下室） |
| **速记** | 玩家笔记本，可加「线索标签」、自动保存，预填案件线索与疑问 |

### 4.4 输入区
- **语音输入为主**（MVP 直接实现），**文字输入为辅**并存；当前 HTML 原型只画了文字框（`⌘+Enter`），语音入口待补。
- 快捷短语按钮：`🔍侦查周围`、`🗣️与队友商量`、`📋查看线索`、`💼检查装备`。

### 4.5 地图渲染
- 优先复用**模组内 / COC 世界已有的地图**素材渲染；无现成图时**可选 AI 生图**（见 D5）。

## 五、关键决策（2026-07-09 确认）

> 上一版这里是「待确认」，本节为逐条拍板结果；对原型的影响已回填到二/三/四节。

| # | 议题 | 决策 | 说明 / 影响 |
|---|---|---|---|
| **D1** | **输入方式** | **语音为主、文字为辅**；**MVP 直接做语音** | 为模拟真实 COC 桌面口述体验。语音与文字并存，进同一消息流。原型需补语音入口。 |
| **D2** | **选模组** | **支持玩家自导入模组**；当前内置《古宅幽影》仅为**跑通全流程测试**用 | 需补「S3.5 选模组」屏（内置 + 导入）。模组是**可插拔内容包**，与「世界观→模组」两级抽象对应。 |
| **D3** | **车卡** | **MVP 必做，玩家手动车卡**；**后续把 COC 全部角色卡/职业/技能纳入** | 车卡不是可选项。数据模型需覆盖完整 COC 角色卡结构，为后续扩充留空间。 |
| **D4** | **AI 补位** | **不纳入 MVP** | S6 大厅去掉「AI 调查员自动补位」；玩家不足时不自动补 AI。 |
| **D5** | **地图** | 优先用**模组内 / COC 世界已有地图**渲染；无现成图时**可选 AI 生图** | 不做纯文字地点列表兜底为主，地图是有图的可视化；AI 生图作为补充手段。 |
| **D6** | **多人 scope（架构级）** | **多人-薄进 MS1**：2+ 真手机进同一房间、WS 公共广播 + 私密定向下发、**回合制**；不做生产级实时同步与并发抢话仲裁 | **推翻**旧架构「多人同步降级 MS2」；理由：权限泄底(最高风险)只有 ≥2 客户端才测得出。详见 [[架构设计-整体多视图]] ADR-1/ADR-2。 |
