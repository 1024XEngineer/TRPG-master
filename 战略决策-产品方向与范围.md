# MS1 战略决策：AI 跑团主持人 · 产品方向与范围

版本：v1.0

日期：2026-07-07

作者：高俊周

> 本文是 MS1 的战略决策文档——回答"做什么、为谁做、不做什么"。各 Proposal 以此为基础，不再重复叙述。

---

## 1. 动机 / 用户故事

跑团爱好者面临共同困境：**想玩，但没人愿意当主持人（KP）**。CoC 跑团对 KP 要求极高——需熟记规则书、即兴恐怖叙事、公正裁决检定、管理多个角色的 SAN 值与私密线索。大量多人小团体"有玩家、无 KP"。

一句话：**用 AI 替代真人 KP，降低跑团门槛，让"有人就能跑"。**

## 2. 目标用户

想跑团但找不到真人 KP 的新手玩家，或无人愿当主持人的跑团爱好者。多人小团体，有社交需求，线下或远程均可。

## 3. 现有做法及其不足

| 现有做法 | 不足 |
|----------|------|
| 真人 KP 主持 | KP 稀缺，学习成本高，一场 3-4 小时精力消耗极大 |
| 文字跑团（QQ / Discord + 骰子机器人） | 无结构化规则判定，SAN/战斗靠人工，无私密信息分发 |
| 跑团平台网页（枭熊 等） | 偏工具属性，有骰子有角色卡，**无 AI 叙事**，仍需真人 KP |
| AI 聊天app（星野 等） | 单人体验，不支持多人房间、角色分离、私密线索；AI 易忘规则 |
| 通用 VTT（Foundry VTT / Roll20） | 配置复杂，面向硬核玩家，手机端体验差，无 AI 主持 |

**核心空白**：手机端 + AI 主持 + 多人房间 + 规则硬编码 + 私密信息分发的完整产品不存在——本质上，这是一个经过产品融合实现的蓝海产品。

## 4. 本期范围（MS1，第 1-2 周）

MS1 目标是**方向成立、骨架串联**，不是实现全部功能。

| # | 交付物 | 说明 |
|---|--------|------|
| 1 | 战略决策 + 产品 Proposal | 本文 + 三份窄提案，团队评审通过 |
| 2 | 架构骨架入库 | 模块拆分、接口定义、数据模型——代码入库、能编译 |
| 3 | 前端可交互原型 | 核心屏幕串联，mock 数据驱动 |

核心屏幕流程：
```
加入房间 → 选择规则（CoC）→ 导入预设模组 → 背景介绍 → 角色卡创建
→ 等候游戏开启 → AI 主持 → 各玩家输入行动 → AI 推动剧情 → 复盘
```

## 5. 明确不做（MS1）

- D&D / 狼人杀 / 血染钟楼 / 剧本杀（仅 CoC 7th）
- 纯粹平台类 / 角色扮演助手产品
- 付费与商城
- 语音输入（文字优先）
- AIGC 图片生成（延后）
- AI NPC / AI 补位玩家（延后）
- 赛后复盘（延后）
- 上传自定义模组（MS1 用预设模组）

> v1 功能另开提案。技术选型（前端框架、通信协议、AI 模型供应商）属架构设计范畴。

## 6. 基本概念与信息结构

以下定义供所有 Proposal 引用，是产品层面的概念定义，非数据库 schema。

### 领域核心概念

- **房间（Room）**：一次跑团会话容器，包含 1 个 KP（AI）+ N 个玩家
- **角色卡（Character Sheet）**：玩家化身，含属性、技能、SAN、HP、物品
- **模组（Module）**：预设剧本，含场景图、NPC、物品、事件、结局
- **场景（Scene）**：模组中的地点节点，含描述、出口、在场 NPC、可发现物品
- **检定（Skill Check）**：AI 叙事触发、规则引擎执行的 D100 判定
- **私密信息（Private Info）**：仅特定玩家可见的线索，AI 生成、定向推送

### 角色卡

```json
{
  "id": "p1",
  "name": "杰克·布朗",
  "occupation": "私家侦探",
  "age": 32,
  "str": 50, "con": 60, "siz": 55, "dex": 65,
  "app": 60, "int": 70, "pow": 65, "edu": 70,
  "hp": 11, "maxHp": 11,
  "san": 65, "maxSan": 99,
  "luck": 55,
  "mp": 14, "maxMp": 14,
  "skills": {
    "spot_hidden": 65, "listen": 50,
    "library_use": 60, "persuade": 45,
    "fighting_brawl": 35, "firearms": 30,
    "stealth": 40, "first_aid": 40,
    "occult": 25, "psychology": 30
  },
  "inventory": ["手电筒", "笔记本", "左轮手枪(.38)", "急救包"]
}
```

- 8 项基础属性，值域 0-99
- 衍生属性（HP/SAN/LUCK/MP）由规则引擎自动计算
- 技能列表不固定，由模组和职业决定
- 物品为字符串数组，不做物品系统

### 场景

```json
{
  "id": "whitley_foyer",
  "name": "惠特利旧宅 · 门厅",
  "description": "吊灯挂满蛛网，壁画人物眼睛被涂黑",
  "type": "static",
  "exits": ["whitley_living", "whitley_stairs", "whitley_basement"],
  "npcs": [],
  "items": ["floor_footprints"]
}
```

- `type: "static"` 进入不触发事件；`"trigger"` 进入即触发
- `exits` 定义可移动方向，场景间形成有向图

### NPC

```json
{
  "id": "prof_armitage",
  "name": "阿米蒂奇博士",
  "location": "miskatonic_library",
  "dialogue_triggers": {
    "greeting": "你们来了。感谢上帝。情况比我想的还要糟。",
    "ask_about_book": "那本书…它不该存在于任何图书馆里。"
  },
  "knowledge": ["basement_key_location"]
}
```

- `dialogue_triggers`：关键词 → 对话模板，AI 以此展开叙事
- `knowledge`：NPC 掌握的线索标签，决定 AI 能透露什么
